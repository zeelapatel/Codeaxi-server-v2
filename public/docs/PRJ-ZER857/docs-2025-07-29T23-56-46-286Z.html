Overview

Welcome to the API documentation. Here you will find all the information you need to integrate with our API.

Auth

Authentication is required for all endpoints. Use your API key to authenticate requests.

API Reference

Get Project

URL: /projects/:id

Method: GET

Parameters:

  • id (string) - The ID of the project.

Response:

{
  "id": "string",
  "name": "string",
  "description": "string",
  "projectContext": "object",
  "generatedApiDocs": "array"
}

Example:

fetch('/projects/123', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
}).then(response => response.json())
  .then(data => console.log(data));

Delete Project

URL: /projects/:id

Method: DELETE

Parameters:

  • id (string) - The ID of the project.

Response: 204 No Content

Example:

fetch('/projects/123', {
  method: 'DELETE',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY'
  }
}).then(response => {
  if (response.status === 204) {
    console.log('Project deleted successfully');
  }
});

Errors

Common error responses include 404 for not found and 500 for server errors.

Rate Limits

API requests are rate-limited to ensure fair usage. Please refer to your API plan for specific limits.

SDKs

SDKs are available for various programming languages to simplify integration.
